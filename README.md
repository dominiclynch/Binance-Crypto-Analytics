# Binance Crypto Analytics

Thin API layer that combines live candle data with ML-generated trade probabilities into actionable trading signals.

## What It Does

- Proxies latest 1-minute candles from the upstream Data Service
- Reads ML probability files generated by the ML engine
- Synthesises trade recommendations: BUY_LONG (prob >= 0.6) or SHORT (prob <= 0.4)
- Serves combined analysis via REST endpoints for dashboards and execution bots

## API Endpoints

| Endpoint | Description |
|----------|-------------|
| `GET /api/analysis` | Combined candle data + ML signals with trade recommendations |
| `GET /api/latest` | Proxy latest candles from Data Service |
| `GET /api/signals` | Pre-computed signal file from ML engine |
| `GET /api/status` | Service health |
| `GET /metrics` | Prometheus metrics |

## Architecture

```
Data Service (candles)  --\
                           -->  Analytics API  -->  Dashboard / Execution Bot
ML Engine (probabilities) --/
```

## Tech Stack

- **Python** — FastAPI, Uvicorn, aiohttp, Pydantic
- **Monitoring** — Prometheus metrics
- **Config** — Environment variables for upstream service URLs

## Setup

```bash
pip install -r requirements.txt
export DATA_SERVICE_URL=http://your-data-service:8001
python -m uvicorn main:app --host 0.0.0.0 --port 8080
```
